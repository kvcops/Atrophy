## 🚨 The Silent Problem

<img align="right" width="180" src="https://raw.githubusercontent.com/Tarikul-Islam-Anik/Animated-Fluent-Emojis/master/Emojis/Smilies/Anxious%20Face%20with%20Sweat.png" alt="Anxious"/>

> *"We all felt it happen. At first, AI was just a faster autocomplete — a way to skip boilerplate. Then came the era of zero-friction development."*

You are **shipping features faster than ever.** Your PRs are getting merged. Your velocity charts look incredible.

But in the quiet moments between prompts, there's a creeping realization:

```
❓ When was the last time you wrote a complex regex from scratch?
❓ How long does it take you to implement a SQL window function now?
❓ Could you reverse a binary tree today — on a whiteboard, alone?
```

You're outsourcing the friction. **And friction is where learning happens.**

Your fundamental, raw engineering skills are slowly, silently **atrophying** — and nobody is tracking it.

## 💡 The Solution

<img align="left" width="160" src="https://raw.githubusercontent.com/Tarikul-Islam-Anik/Animated-Fluent-Emojis/master/Emojis/Smilies/Alien%20Monster.png" alt="Atrophy Monster"/>

&nbsp;&nbsp;&nbsp;&nbsp;**`atrophy`** is the honest mirror you didn't know you needed.

&nbsp;&nbsp;&nbsp;&nbsp;It quietly analyzes your **git commit history — 100% locally** — to measure exactly which skills you're still exercising and which ones you've silently handed over to AI.

&nbsp;&nbsp;&nbsp;&nbsp;It doesn't judge. It doesn't report to your boss. It doesn't stop you from using AI.

&nbsp;&nbsp;&nbsp;&nbsp;It tracks your **Coding Fingerprint™**, and delivers hyper-targeted mini-challenges designed to rebuild the specific muscles you're losing — one commit at a time.

## ⚡️ Quickstart

> Get from zero to insights in **under 2 minutes.**

```bash
# ① Initialize — connect Atrophy to your repository (one-time setup)
atrophy init

# ② Scan — analyze your recent git history & classify every commit
atrophy scan

# ③ Dashboard — launch the interactive, animated terminal UI
atrophy dashboard

# ④ Challenge — generate hyper-personalized exercises for rusty skills
atrophy challenge --generate

# ⑤ Digest — export a beautiful weekly report to your journal
atrophy digest --open
```

## 🔬 The 10 Disciplines

Atrophy maps every commit across **10 canonical engineering disciplines** — your universal skill genome:

<div align="center">

| &nbsp; | Discipline | What It Tracks | Detection Signals |
|:---:|:---|:---|:---|
| 🧠 | **Algorithms** | Data structures, recursion, dynamic programming | Abstract graph/tree traversals, custom sort logic, memoization |
| 🔣 | **Regex Parsing** | Complex text pattern matching | Capture groups, lookaheads, backreferences, non-trivial patterns |
| 🗄️ | **SQL / Databases** | Query optimization, schema architecture | Window functions, CTEs, recursive queries, complex JOINs |
| 🏗️ | **API Design** | System design, abstraction, contracts | Interface definitions, dependency injection, versioning |
| 🛡️ | **Security** | Safe encoding, cryptography, hardening | Parameterized queries, CSRF, input sanitization, JWT handling |
| 🚦 | **Async / Concurrency** | Threading, multiprocessing, event loops | Mutexes, concurrent queues, task groups, semaphores |
| 🧪 | **Testing** | Mocks, fixtures, property-based tests | Custom test harnesses, deep mocking, hypothesis-style tests |
| 📦 | **DevOps / SysAdmin** | CI/CD pipelines, containers, infrastructure | Complex bash pipelines, Docker multi-stage, workflow logic |
| 🎨 | **CSS / UI / UX** | Web styling, layout systems, animation | CSS Grid, keyframes, fluid typography, container queries |
| 🔧 | **Tooling** | Build scripts, parsers, code generation | Custom parsers, AST transformations, CLI frameworks |

## ❓ FAQ

<details>
<summary><b>🔒 Does my source code ever leave my machine?</b></summary>
<br>

**No.** `atrophy` runs **100% locally** by default. Your git history never leaves your physical hardware.

The *only* exception: if you configure a cloud LLM provider (OpenAI, Anthropic) to generate personalized challenges — in which case **short, highly sanitized code snippets** are sent solely for exercise generation. Your full source is never transmitted.

Connect **Ollama** to run the entire backend fully locally and privately — zero cloud, zero trust required.

</details>

<details>
<summary><b>🤔 Can I use Atrophy entirely offline without an API key?</b></summary>
<br>

**Absolutely.** `atrophy init` lets you skip LLM configuration entirely. You'll still get:

- ✅ Full Coding Fingerprint tracking
- ✅ Animated terminal dashboard
- ✅ Weekly digest exports
- ✅ Heuristic-based AI detection
- ❌ AI-generated personalized challenges (requires LLM)

Want AI features with zero cloud? Connect a local **Ollama** instance and get everything.

</details>

<details>
<summary><b>🎯 How accurate is the AI detection?</b></summary>
<br>

The baseline detector uses **5 statistical heuristics** (commit velocity, burstiness, entropy, formatter-presence, and message depth) combined with your **personal developer baseline** — the cadence and patterns unique to how *you* code.

It is highly accurate for identifying AI-assisted commits, but no algorithm is perfect. Atrophy is designed to be an **honest mirror**, not a verdict machine. Use it as a reflection tool, not a grade.

</details>

<details>
<summary><b>⚙️ Which editors and version control systems are supported?</b></summary>
<br>

Atrophy works with any **git repository**, regardless of editor or IDE. Whether you use VS Code, Neovim, JetBrains, or bare terminal — if it's committed to git, Atrophy can analyze it.

Support for **Mercurial** and **SVG** repositories is on the roadmap.

</details>

<details>
<summary><b>📅 How often should I run <code>atrophy scan</code>?</b></summary>
<br>

For the most accurate fingerprint, we recommend running `atrophy scan` **at least once a week** — or better yet, hooking it into a simple cron job or git hook so it runs automatically after each session. The more data Atrophy has, the sharper its recommendations become.

</details>
